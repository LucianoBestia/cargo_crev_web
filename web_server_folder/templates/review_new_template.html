Prerequisites to write new crev reviews for a crate version:
- on your developer machine install $ cargo install cargo-crev,
- on GitHub or GitLab fork the repo https://github.com/crev-dev/crev-proofs 
- clone it to your local machine
- create your personal crev ID $ cargo crev id new
Everything is encrypted, so be careful with the secrets, keys, passphrase,... There's no way to recover them.
Read the Get Start Guide to know better. 

The new crev review must be written with the command
$ cargo crev crate review -u --skip-activity-check reader_for_microxml 1.1.11 
It opens the VIM editor. 
This is the important part: When saved, the review is encrypted automatically ! 
But writing in VIM is not easy for everybody. 
Vim has "modes": Normal, Insert, Command,... Confusing. 
By the way when you panic in VIM, type Esc and :q or :q! to exit or quit VIM. 
There is also the possibility to write in your favorite editor, that is much better, but still not the best user experience. 
The browser cannot finish the review with encryption. So we need a combination of all: browser, bash and VIM.

- on this webpage write a review 
- Export to yaml
- copy the yaml text, 
- run the command $ cargo crev crate review -u --skip-activity-check reader_for_microxml 1.1.11.
- VIM will open in Normal mode
- delete the first cc 10 lines with repeating the VIM shortcut dd for delete line 
- then in VIM type i to go into VIM Insert Mode 
- then mouse right click to paste 
- then Esc to go to VIM Normal mode
- type : to go into VIM command mode
- then type :wq to write and quit 
- now the encryption will kick in 
- Done. Easy?!
- check your new review locally cargo crev repo query review reader_for_microxml
- publish (push to GitHub) cargo crev publish

Now everybody can fetch and view your crev review.
On this web site the new review will be fetched once in the next hour.
